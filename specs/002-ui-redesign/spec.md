# Feature Specification: Virtual Coach App UI Redesign with Matcha Green Theme

**Feature Branch**: `002-ui-redesign`  
**Created**: 2025-11-15  
**Status**: Draft  
**Input**: User description: "UI Design - 請以 UI/UX 資深設計師的角度,重新設計一個以網頁應用為主的 UI 介面,整體配色我想選用舒服的淡抹綠色做為色彩基調。若有需要,可以進行功能上的改動。最後請檢查所有功能整合的完整性,以確保所有功能可以正常運行"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 視覺煥然一新的偏好設定體驗 (Refreshed Preference Setting Experience) (Priority: P1)

學員開啟應用程式後,看到以淡抹綠色為主調的溫和視覺介面,透過卡片式設計和視覺圖示快速理解各個偏好選項,選擇過程直覺流暢,配色柔和不刺眼,整體感受如同置身於寧靜的健身空間。

**Why this priority**: 偏好設定是使用者與應用的第一接觸點,良好的視覺設計和使用體驗直接影響使用者對產品的第一印象和持續使用意願。這是 UI 改版的核心價值所在。

**Independent Test**: 可透過以下步驟獨立測試:開啟應用 → 檢視主畫面配色是否為淡抹綠色調 → 驗證所有偏好選項都有清晰的視覺呈現(圖示、卡片、標籤)→ 測試選擇各選項時的視覺回饋 → 確認整體視覺和諧且易於操作。

**Acceptance Scenarios**:

1. **Given** 學員首次開啟應用程式,**When** 載入主畫面,**Then** 應看到以淡抹綠色(#C8E6C9 ~ #E8F5E9 色系)為主調的介面,背景使用柔和漸層,主要操作區域使用白色或淡綠色卡片,文字清晰可讀,整體視覺溫和舒適。

2. **Given** 學員正在設定偏好,**When** 瀏覽「運動目標」選項,**Then** 每個選項應以視覺卡片呈現,包含代表性圖示(增肌:肌肉圖示、減脂:火焰圖示、塑形:身型圖示),卡片使用淡綠色邊框,選中時以深綠色(#66BB6A)高亮顯示,並有平滑的動畫過渡效果。

3. **Given** 學員正在選擇器材,**When** 點擊或懸停在器材選項上,**Then** 系統應提供即時視覺回饋:未選中時為淡綠色輪廓,選中時填充淡綠色背景並顯示勾選圖示,懸停時輕微放大(scale 1.05),動畫流暢自然(transition 0.3s ease)。

4. **Given** 學員正在調整訓練時長,**When** 使用時長選擇器(滑桿或按鈕組),**Then** 選擇器應使用淡綠色調,滑桿軌道為淺綠色(#E8F5E9),滑桿手柄為中綠色(#81C784),已選擇的軌道區段為深綠色(#66BB6A),數值變化時即時更新且動畫流暢。

5. **Given** 學員完成所有偏好設定,**When** 點擊「生成課表」按鈕,**Then** 按鈕應使用吸睛但和諧的深綠色(#66BB6A),文字清晰(白色),按鈕有適當的圓角和陰影,點擊時有按壓效果(scale 0.98),載入時顯示綠色系的 loading 動畫(圓形 spinner 或進度條)。

### User Story 2 - 清晰美觀的訓練課表展示 (Clear and Beautiful Workout Plan Display) (Priority: P2)

學員生成課表後,看到以卡片式佈局呈現的訓練計畫,每個運動項目都有清晰的視覺層次、易讀的資訊排版和吸引人的配色,整體視覺與偏好設定頁面保持一致,讓學員能快速瀏覽並理解課表內容。

**Why this priority**: 課表展示是學員確認訓練內容的關鍵頁面,清晰的視覺設計能幫助學員快速理解課表結構,提升對訓練的掌控感和信心。這是偏好設定之後的自然延伸。

**Independent Test**: 可透過以下步驟獨立測試:生成一份包含 5-8 個運動的課表 → 驗證課表頁面配色與主題一致 → 檢查每個運動卡片的資訊排版清晰度 → 測試卡片的互動效果(展開/收合)→ 確認操作按鈕(開始訓練、重新生成)視覺突出且易於點擊。

**Acceptance Scenarios**:

1. **Given** 學員已成功生成一份課表,**When** 顯示課表頁面,**Then** 頁面應以垂直捲動的卡片列表呈現,每張卡片使用白色背景和淡綠色邊框,卡片間距適當(16-24px),整體佈局整潔有序,頂部顯示課表摘要(總時長、運動數量)使用綠色系標籤。

2. **Given** 學員正在瀏覽課表,**When** 查看每個運動卡片,**Then** 卡片應包含:運動名稱(大字體、深色)、器材標籤(綠色圖示+文字)、時長標籤(時鐘圖示+秒數)、難度標籤(星級圖示、綠色系),卡片右側顯示運動縮圖(如有影片則顯示播放圖示),整體資訊層次分明,一眼即可識別關鍵資訊。

3. **Given** 學員正在查看某個運動的詳細資訊,**When** 點擊卡片展開詳情,**Then** 卡片應平滑展開(動畫 0.3s ease),顯示完整的動作說明、注意事項(使用淺綠色背景的提示框)、示範影片縮圖,展開後的卡片有視覺區隔(輕微陰影增強),再次點擊可收合。

4. **Given** 學員確認課表滿意,**When** 準備開始訓練,**Then** 頁面底部(或頂部)應固定顯示操作按鈕區:「開始訓練」按鈕使用深綠色(#66BB6A)、大尺寸、全寬度或居中突出,「重新生成」按鈕使用淡綠色輪廓樣式(outline),兩按鈕間距適當,點擊區域足夠大(至少 44x44px)。

5. **Given** 學員想重新生成課表,**When** 點擊「重新生成」按鈕,**Then** 系統應顯示確認對話框(使用綠色系主題),對話框背景為淡綠色半透明遮罩,內容卡片為白色,確認按鈕為深綠色,取消按鈈為灰色輪廓,對話框有淡入淡出動畫。

### User Story 3 - 沉浸式綠色系訓練播放器 (Immersive Green-Themed Training Player) (Priority: P3)

學員開始訓練後進入全螢幕播放器,整體配色延續淡綠色主題但更專注於訓練內容,背景使用深綠漸層營造專注氛圍,倒數計時器和控制按鈕清晰可見,視覺元素簡約不干擾訓練,整體感受如同專業訓練環境。

**Why this priority**: 訓練播放器是學員實際執行訓練的核心場景,良好的視覺設計能幫助學員保持專注,但這屬於進階體驗優化,在完成前兩個核心介面的改版後實施。

**Independent Test**: 可透過以下步驟獨立測試:開始訓練進入播放器 → 驗證播放器全螢幕顯示且配色符合綠色主題 → 檢查倒數計時器的視覺效果(圓形進度條、數字倒數)→ 測試控制按鈕(暫停、跳過、退出)的視覺清晰度和互動回饋 → 確認運動切換時的過渡動畫流暢。

**Acceptance Scenarios**:

1. **Given** 學員點擊「開始訓練」,**When** 進入播放器模式,**Then** 播放器應全螢幕顯示,背景使用深綠色到黑色的漸層(#2E7D32 → #1B5E20),營造沉浸感,內容區域(影片、資訊)使用半透明白色或淺綠色背景,文字清晰可讀(白色或深灰色),整體視覺專業且不刺眼。

2. **Given** 播放器正在執行訓練,**When** 顯示倒數計時器,**Then** 計時器應使用大型圓形進度條設計,外環為綠色(#66BB6A)隨時間減少而逐漸填滿,內部顯示剩餘秒數(大字體、白色或淺色),進度條有平滑動畫,每秒更新時無卡頓,視覺清晰醒目。

3. **Given** 播放器正在播放運動影片,**When** 學員查看運動資訊,**Then** 畫面上方顯示運動名稱(大字體、白色、有淺色背景增強可讀性),下方顯示動作說明(中等字體、使用半透明淺綠色背景框),左上角顯示當前進度(如「3 / 8」運動序號、綠色標籤),資訊不遮擋影片核心內容。

4. **Given** 播放器正在執行,**When** 學員需要控制訓練,**Then** 畫面下方(或側邊)顯示半透明控制列,包含「暫停/繼續」按鈕(圓形、綠色)、「跳過」按鈕(圓形、淺綠色)、「退出」按鈕(圓形、灰色),按鈕有圖示,懸停時顯示提示文字,點擊時有視覺回饋(縮放或顏色變化),控制列在無互動 3 秒後自動淡出,移動滑鼠時重新顯示。

5. **Given** 播放器完成所有運動,**When** 顯示訓練完成畫面,**Then** 畫面應使用慶祝性視覺設計:背景使用明亮的綠色漸層(#81C784 → #66BB6A),中央顯示完成圖示(勾選或獎杯、大尺寸、動畫效果),顯示訓練總結(總時長、完成運動數、使用綠色卡片呈現),底部提供「返回主畫面」按鈕(白色背景、綠色文字)和「分享成績」按鈕(可選、綠色背景)。

6. **Given** 播放器運動切換時,**When** 從一個運動切換到下一個,**Then** 應有流暢的過渡動畫:前一運動淡出(fade-out 0.5s),短暫顯示過渡畫面(綠色背景、顯示「下一個運動」文字和預覽圖示、持續 2 秒),然後新運動淡入(fade-in 0.5s),整體過渡自然不突兀。

### User Story 4 - 響應式設計與多裝置適配 (Responsive Design & Multi-Device Adaptation) (Priority: P4)

無論學員使用手機、平板或桌面裝置,應用程式都能提供最佳化的視覺體驗,在不同螢幕尺寸下自動調整佈局、字體大小和互動元素尺寸,綠色主題在所有裝置上保持一致,操作流暢不受裝置限制。

**Why this priority**: 響應式設計確保應用能服務更廣泛的使用者群體,但在 MVP 改版階段,優先完成核心介面的視覺改版,響應式優化可在後續迭代完善。

**Independent Test**: 可透過以下步驟獨立測試:使用開發者工具切換不同裝置模擬 → 在手機尺寸(320-428px)測試所有頁面 → 在平板尺寸(768-1024px)測試 → 在桌面尺寸(1280px+)測試 → 驗證佈局自動調整、文字可讀、按鈕可點擊、配色一致。

**Acceptance Scenarios**:

1. **Given** 學員使用手機(螢幕寬度 < 768px),**When** 開啟偏好設定頁面,**Then** 介面應採用單欄佈局,偏好選項卡片垂直排列,每個選項區域(目標、器材、時長、難度)佔滿螢幕寬度(減去邊距),字體大小適當(至少 16px),按鈕觸控區域足夠大(至少 44x44px),整體視覺與桌面版一致。

2. **Given** 學員使用平板(螢幕寬度 768-1024px),**When** 查看課表頁面,**Then** 介面應採用兩欄佈局(可選)或保持單欄但增加卡片寬度,運動卡片橫向佔用更多空間,資訊排版更寬鬆,字體略大於手機版,保持綠色主題一致性。

3. **Given** 學員使用桌面裝置(螢幕寬度 ≥ 1280px),**When** 瀏覽任何頁面,**Then** 介面應使用居中佈局,最大寬度限制在 1200-1440px,左右留白,內容區域寬敞,偏好設定可採用卡片網格佈局(2x2 或 3x2),課表使用兩欄佈局(左側列表、右側詳情),播放器使用橫向佈局(左側影片、右側資訊)。

4. **Given** 學員在任何裝置上使用播放器,**When** 進入全螢幕訓練模式,**Then** 播放器應自動適應螢幕方向和尺寸,手機橫向時影片全螢幕,控制列自動調整位置,倒數計時器和資訊調整大小,觸控按鈕在小螢幕上仍然易於點擊。

### Edge Cases

- **當瀏覽器不支援現代 CSS 特性時**:系統應提供降級方案,確保基本功能可用,配色保持綠色主題但不使用複雜動畫或漸層,顯示提示建議使用現代瀏覽器獲得最佳體驗。

- **當使用者啟用深色模式偏好時**:系統可選擇性支援深色模式,將淡綠色主題調整為深綠色基調(#2E7D32 背景 + #81C784 強調色),或保持原淡綠色主題但在深色模式下提供切換開關。

- **當使用者使用高對比度模式或輔助功能時**:系統應確保所有文字和互動元素符合 WCAG AA 標準(對比度至少 4.5:1),綠色與白色/深灰色的對比度應足夠,提供足夠的視覺反饋(不僅依賴顏色)。

- **當使用者快速連續點擊按鈕時**:系統應防止重複提交,按鈕在處理中時應顯示 loading 狀態並禁用,避免多次觸發相同操作,視覺上應有明確的載入指示(綠色 spinner 或進度條)。

- **當網路載入緩慢或圖示資源失敗時**:系統應顯示佔位符(skeleton screen 或淡綠色漸層背景),圖示載入失敗時使用文字替代或預設圖示,確保介面不會出現空白或破損,保持視覺完整性。

- **當使用者縮放瀏覽器頁面時**:系統應支援 200% 縮放而不破壞佈局,文字和互動元素應隨之放大,保持可讀性和可操作性,滾動條和溢出內容應正確處理。

- **當使用者使用極小或極大螢幕時**:極小螢幕(< 320px)應顯示提示建議使用更大螢幕以獲得最佳體驗,極大螢幕(> 2560px)應限制內容最大寬度並居中顯示,避免元素過度拉伸。

## Requirements *(mandatory)*

### Functional Requirements

#### Epic 1: 視覺設計系統 (Visual Design System)

- **FR-001**: 系統必須定義並應用一套完整的綠色系色彩規範,包含主色(#66BB6A)、淺色(#C8E6C9 ~ #E8F5E9)、深色(#2E7D32 ~ #1B5E20)、強調色(#81C784)和中性色(灰階),所有 UI 元素必須使用此色彩系統。

- **FR-002**: 系統必須定義字體階層系統,包含標題(32-48px、粗體)、副標題(24-28px、半粗)、正文(16-18px、正常)、輔助文字(12-14px、正常),所有文字必須使用無襯線字體(如 Inter、Noto Sans TC)確保可讀性。

- **FR-003**: 系統必須定義間距系統,使用 4px 基礎單位(4, 8, 12, 16, 24, 32, 48, 64px),所有元素間距、內邊距、外邊距必須遵循此規範,確保視覺一致性。

- **FR-004**: 系統必須定義圓角規範,卡片使用中等圓角(12-16px)、按鈕使用小圓角(8-12px)、標籤使用大圓角(20-24px),輸入框使用小圓角(6-8px),確保整體視覺柔和。

- **FR-005**: 系統必須定義陰影系統,使用三層陰影(小陰影:卡片靜態、中陰影:卡片懸停、大陰影:對話框和浮動元素),陰影顏色使用綠色系半透明(rgba(46, 125, 50, 0.1-0.3)),營造柔和的深度感。

- **FR-006**: 系統必須定義動畫規範,過渡動畫使用 ease-in-out 曲線、持續時間 0.2-0.5s,載入動畫使用綠色系、圓形或線性進度條,所有動畫必須流暢不卡頓(60fps)。

#### Epic 2: 偏好設定介面改版 (Preference Setting UI Redesign)

- **FR-007**: 偏好設定頁面必須使用卡片式佈局,每個偏好類別(目標、器材、時長、難度)使用獨立卡片,卡片背景為白色或淡綠色(#F1F8E9),邊框使用淡綠色(#C8E6C9),卡片間距 16-24px。

- **FR-008**: 每個偏好選項必須包含視覺圖示,使用 SVG 格式、綠色系填充或輪廓,圖示大小 32-48px,與文字標籤垂直或水平排列,選中時圖示顏色變深(#66BB6A)。

- **FR-009**: 偏好選項的選中狀態必須有明確視覺反饋:未選中時為淡綠色輪廓(border 2px solid #C8E6C9)、選中時填充淡綠色背景(#E8F5E9)並顯示綠色勾選圖示(#66BB6A)、懸停時輕微放大(transform scale 1.05)。

- **FR-010**: 時長選擇器必須提供清晰的視覺化呈現,可使用滑桿(slider)或按鈕組(button group),滑桿軌道為淺綠色(#E8F5E9)、已選擇區段為深綠色(#66BB6A)、手柄為中綠色(#81C784)並有陰影,數值即時顯示在滑桿上方或旁邊。

- **FR-011**: 「生成課表」按鈕必須視覺突出,使用深綠色背景(#66BB6A)、白色文字、中等圓角(8-12px)、中等陰影,按鈕寬度至少佔畫面 50%(手機)或固定寬度 240-320px(桌面),點擊時有按壓效果(scale 0.98)。

- **FR-012**: 載入生成狀態時必須顯示綠色系 loading 動畫,可使用圓形 spinner(旋轉動畫)或線性進度條,spinner 顏色為深綠色(#66BB6A)到中綠色(#81C784)漸變,動畫流暢(1-2s 循環),並顯示提示文字「正在生成您的專屬訓練計畫...」。

#### Epic 3: 訓練課表介面改版 (Workout Plan UI Redesign)

- **FR-013**: 課表頁面必須使用垂直捲動卡片列表,每個運動項目使用獨立卡片,卡片背景為白色、邊框為淡綠色(#C8E6C9 1px solid)、圓角 12-16px、小陰影,卡片間距 12-16px。

- **FR-014**: 課表摘要資訊必須顯示在頁面頂部,使用橫向排列的標籤組,包含總時長標籤(時鐘圖示 + 數值、淺綠色背景 #E8F5E9)、運動數量標籤(數字圖示 + 數值)、難度標籤(星級圖示),標籤使用大圓角(20-24px)、內邊距 8-12px。

- **FR-015**: 每個運動卡片必須包含清晰的資訊層次:左側顯示運動名稱(大字體 18-20px、粗體、深灰色 #212121)、標籤組(器材、時長、難度、小尺寸 12-14px、綠色系圖示),右側顯示運動縮圖(圓角正方形 80-100px)或影片播放圖示。

- **FR-016**: 運動卡片的標籤必須使用視覺圖示+文字組合,器材標籤使用啞鈴/徒手圖示(綠色)、時長標籤使用時鐘圖示、難度標籤使用星級圖示(1-3 星、填充綠色),標籤使用小圓角(6-8px)、淺綠色背景(#F1F8E9)、內邊距 4-8px。

- **FR-017**: 運動卡片必須支援展開/收合功能,點擊卡片時平滑展開(transition 0.3s ease)顯示完整說明和注意事項,展開區域使用淺綠色背景(#F1F8E9)、內邊距 16px,再次點擊收合,展開時卡片陰影加深。

- **FR-018**: 操作按鈕區必須固定在頁面底部或頂部,包含「開始訓練」主按鈕(深綠色 #66BB6A、白色文字、全寬度或居中大尺寸)和「重新生成」次要按鈕(淡綠色輪廓 border 2px solid #81C784、綠色文字),兩按鈕間距 12-16px。

#### Epic 4: 訓練播放器介面改版 (Training Player UI Redesign)

- **FR-019**: 播放器必須全螢幕顯示,背景使用深綠色到黑色漸層(linear-gradient #2E7D32 → #1B5E20),營造沉浸式訓練環境,內容區域使用半透明卡片(rgba(255, 255, 255, 0.95))確保資訊清晰可見。

- **FR-020**: 倒數計時器必須使用大型圓形進度條設計,外環為綠色(#66BB6A、stroke-width 8-12px)隨時間減少逐漸填滿,內部顯示剩餘秒數(48-72px、粗體、白色或淺色),進度條直徑 200-300px(桌面)或 150-200px(手機),動畫平滑(CSS animation)。

- **FR-021**: 運動資訊顯示區必須包含:頂部運動名稱(32-48px、粗體、白色、半透明深色背景增強可讀性)、下方動作說明(16-18px、白色或淺色、半透明淺綠色背景框 rgba(200, 230, 201, 0.3))、左上角進度標籤(「3 / 8」、綠色卡片 #81C784、白色文字)。

- **FR-022**: 播放器控制列必須包含三個圓形按鈕,使用半透明背景(rgba(46, 125, 50, 0.8))、白色圖示、直徑 56-64px,按鈕包含「暫停/繼續」(中央、播放圖示)、「跳過」(右側、快進圖示)、「退出」(左側、返回圖示),懸停時顯示提示文字(白色、小圓角背景)。

- **FR-023**: 控制列必須支援自動隱藏,無互動 3 秒後淡出(opacity 0、transition 0.5s ease),滑鼠移動或觸控時重新顯示(opacity 1),確保不干擾訓練但隨時可用。

- **FR-024**: 訓練完成畫面必須使用慶祝性視覺,背景為明亮綠色漸層(#81C784 → #66BB6A)、中央顯示完成圖示(勾選或獎杯、128-160px、綠色轉白色、淡入+彈跳動畫)、訓練總結卡片(白色背景、綠色強調元素)、操作按鈕(「返回主畫面」、「分享成績」)。

- **FR-025**: 運動切換過渡必須流暢,前一運動淡出(fade-out 0.5s)→ 過渡畫面(綠色背景 #66BB6A、顯示「下一個運動」文字和預覽、持續 2s)→ 新運動淡入(fade-in 0.5s),過渡畫面包含下一運動名稱、圖示和倒數「3... 2... 1...」。

#### Epic 5: 響應式與互動優化 (Responsive & Interaction Optimization)

- **FR-026**: 系統必須支援三種螢幕尺寸斷點:手機(< 768px)、平板(768-1024px)、桌面(≥ 1024px),使用 CSS media queries 實現響應式佈局,每個斷點下元素自動調整尺寸、間距和排列方式。

- **FR-027**: 手機佈局必須採用單欄垂直佈局,偏好選項卡片佔滿螢幕寬度(減去 16-24px 邊距)、字體至少 16px、按鈕高度至少 44px、觸控區域足夠大,課表卡片全寬顯示、資訊垂直排列。

- **FR-028**: 桌面佈局必須使用居中佈局,內容區最大寬度 1200-1440px、左右留白,偏好設定使用 2x2 或 3x2 網格佈局,課表使用雙欄(左側列表、右側詳情可選),播放器使用橫向佈局(左側影片、右側資訊可選)。

- **FR-029**: 所有互動元素必須提供懸停和激活狀態,懸停時輕微變化(顏色變深、輕微放大 scale 1.05、陰影加深),激活時按壓效果(scale 0.98-0.95)、顏色進一步變深,過渡動畫 0.2-0.3s ease。

- **FR-030**: 系統必須支援鍵盤導航,所有可互動元素可透過 Tab 鍵切換、Enter/Space 鍵激活,焦點狀態有明確視覺反饋(綠色輪廓 outline 2-3px solid #66BB6A、offset 2px),播放器支援空白鍵暫停/繼續、右箭頭跳過、ESC 退出。

- **FR-031**: 所有文字與背景對比度必須符合 WCAG AA 標準(至少 4.5:1),深綠色文字(#2E7D32)在白色背景上、白色文字在深綠色背景上都應滿足標準,提供足夠視覺反饋不僅依賴顏色(如圖示、文字標籤)。

#### Epic 6: 效能與載入優化 (Performance & Loading Optimization)

- **FR-032**: 系統必須優化首次內容繪製時間(FCP),目標 < 1.5 秒,使用程式碼分割(code splitting)、資源預載入(preload)、CSS 內聯關鍵樣式,確保綠色主題樣式優先載入。

- **FR-033**: 圖示和圖片必須優化,使用 SVG 格式圖示(可縮放、檔案小)、WebP 格式圖片(壓縮率高)、懶載入(lazy loading)非首屏圖片,確保頁面載入流暢。

- **FR-034**: 載入狀態必須提供視覺反饋,使用骨架屏(skeleton screen)或綠色系佔位符,避免內容突然跳動(layout shift),載入失敗時顯示友善錯誤訊息和重試按鈕(綠色輪廓)。

- **FR-035**: 動畫必須使用 CSS transform 和 opacity 屬性(硬體加速),避免使用 width/height/left/top 引起重排(reflow),確保動畫流暢(60fps),複雜動畫使用 will-change 提示瀏覽器優化。

### Key Entities

- **Design Token (設計令牌)**: 代表整個設計系統的基礎變數,包含色彩(主色、輔助色、語意色)、字體(字型、大小、行高、粗細)、間距(基礎單位、邊距、內邊距)、圓角、陰影、動畫(持續時間、緩動函數)。所有 UI 元件必須引用 Design Token 確保一致性。以 CSS 變數或 Tailwind 配置形式實現。

- **Component Variant (元件變體)**: 代表同一元件的不同視覺狀態,如按鈕變體(主要/次要/輪廓/文字)、卡片變體(標準/懸停/展開/禁用)、標籤變體(目標/器材/時長/難度)。每個變體有特定的色彩、大小、陰影組合,但遵循統一設計規範。

- **Responsive Breakpoint (響應式斷點)**: 代表不同裝置尺寸的臨界點,包含 mobile(< 768px)、tablet(768-1024px)、desktop(≥ 1024px)。每個斷點定義特定的佈局規則、字體大小、間距倍數,確保所有裝置上的最佳體驗。

- **Animation Curve (動畫曲線)**: 代表不同類型動畫的時間函數,包含 ease-in(漸快、用於離開動畫)、ease-out(漸慢、用於進入動畫)、ease-in-out(兩端慢中間快、用於過渡)、linear(勻速、用於 loading)。每種動畫類型選擇適當曲線確保自然流暢。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 使用者對新介面的視覺滿意度評分達到 4.5/5.0 以上(透過問卷或評分機制收集),相較於舊版提升至少 20%。

- **SC-002**: 偏好設定頁面的完成時間減少至 45 秒以內(從 60 秒縮短),生成課表的操作成功率達到 95% 以上(減少誤操作或困惑)。

- **SC-003**: 課表頁面的資訊查找效率提升 30%,使用者能在 10 秒內找到特定運動的關鍵資訊(名稱、時長、器材),透過眼動追蹤或計時測試驗證。

- **SC-004**: 訓練播放器的沉浸感評分達到 4.3/5.0 以上,85% 的使用者反饋視覺設計有助於保持專注、不會感到分心或視覺疲勞。

- **SC-005**: 新介面的首次內容繪製時間(FCP)保持在 1.5 秒以內,最大內容繪製時間(LCP)保持在 2.5 秒以內,即使增加了動畫和視覺效果也不影響效能。

- **SC-006**: 響應式設計在三種裝置(手機、平板、桌面)上的佈局適配準確率達到 100%,無佈局錯位、文字溢出或按鈕過小問題,透過 Lighthouse 測試和真機測試驗證。

- **SC-007**: 無障礙功能符合 WCAG 2.1 AA 標準,色彩對比度、鍵盤導航、螢幕閱讀器支援都達標,透過 axe DevTools 或 WAVE 工具檢測無重大問題。

- **SC-008**: 使用者在新介面上的任務完成率達到 90% 以上(完成偏好設定 → 生成課表 → 開始訓練 → 完成訓練),相較於舊版提升至少 15%。

- **SC-009**: 綠色主題的品牌識別度達到 80% 以上,使用者能在看到配色後立即聯想到「健康、自然、舒適」等關鍵詞,透過品牌認知測試驗證。

- **SC-010**: 動畫和過渡效果的流暢度評分達到 4.2/5.0 以上,無卡頓或延遲感,所有動畫維持 60fps,透過 Chrome DevTools Performance 檢測幀率。

- **SC-011**: 新介面的使用者留存率(7 日留存)提升至 60% 以上,相較於舊版提升至少 10%,反映良好的視覺體驗能促進持續使用。

- **SC-012**: 設計系統的一致性達到 95% 以上,所有頁面和元件都正確使用 Design Token,無硬編碼顏色或樣式,透過程式碼審查和設計 QA 驗證。
